# Frontend bundle audit (PropertyScope)

## Report

From `frontendV2Hostech/`:

```bash
npm run build:analyze
```

Open `frontendV2Hostech/dist/stats.html` in a browser (generated by `rollup-plugin-visualizer` when Vite runs with `--mode analyze`).

## Targets

- Largest chunks under `src/PropertyScope/` (billing, finance, contracts).
- Prefer **lazy** route-level imports so heavy pages stay out of the initial bundle.

## Lazy loading

Finance routes (`finance/payments`, `finance/ledger`) use `React.lazy` so ledger and payments pages split into separate chunks. Re-run the analyzer after large dependency additions (charts, editors).
